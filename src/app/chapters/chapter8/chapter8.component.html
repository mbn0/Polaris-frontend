ECB: Independent Block Encryption

In ECB mode, each plaintext block Pi is encrypted independently:

Ci = EK(Pi)

This simplicity allows full parallelism but leaks patterns when identical plaintext blocks recur.

Security Warning: Never use ECB for images or structured dataâ€”identical blocks produce identical ciphertext, revealing structure.

CFB: Block Cipher as Stream

CFB turns a block cipher into a self-synchronizing stream cipher. For segment size s:

1. Encrypt Ci-1 (or IV for i=1)
2. XOR leftmost s bits with Pi â†’ Ci
3. Shift Ci into feedback register

Unlike CBC, decryption uses the encryption function, simplifying hardware implementation.

OFB generates a keystream independent of plaintext/ciphertext:

Advantage: Since OFB's keystream is precomputable, it's ideal for high-latency or offline encryption scenarios.

Pure Stream Ciphers

RC4: Byte-Oriented Stream Cipher

RC4 maintains a 256-byte state array and two indices i,j. Each output byte is derived by swapping state entries and XOR'ing with plaintext. Despite its simplicity, RC4's biases render it unsuitable for new applications.

RC4 Components
