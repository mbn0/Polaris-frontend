Why Asymmetric-Key Cryptography?

Asymmetric (public-key) cryptography complements symmetric-key methods by solving the key-distribution problem: you no longer need to share a secret key beforehand. Instead, each user has a public key (widely known) and a private key (kept secret). Messages encrypted with one key can only be decrypted with the other.

Core Advantages

  • Enables digital signatures for authentication and non-repudiation
  • Supports secure key exchange for large-message encryption using hybrid schemes
  • No need for pre-shared secrets between communicating parties

Trade-off: Asymmetric cryptography is computationally more expensive than symmetric cryptography, typically 100-1000x slower.

The Mathematical Backbone

At the heart of every public-key system is a trapdoor one-way function (TOWF): easy to compute in one direction but infeasible to invert without secret "trapdoor" information. Two canonical examples are integer factorization and discrete logarithms.

Optimal Asymmetric Encryption Padding (OAEP)

Raw RSA is vulnerable to deterministic-ciphertext and small-message attacks. OAEP mixes the plaintext with random padding before exponentiation, ensuring each encryption is unique and thwarting chosen-ciphertext exploits.

Digital Signatures & Hybrid Schemes

Digital Signatures

RSA, ElGamal, and ECC support signing by reversing roles: the private key "encrypts" a hash, and anyone verifies with the public key. Signatures provide authentication, integrity, and non-repudiation.

Digital Envelope

To encrypt long messages, generate a random symmetric key K, encrypt data with K, then wrap K via RSA (or ECC). The result is a compact digital envelope combining speed and secure key transport.

Exercise 4: Elliptic-Curve Demo

Use a library (e.g., Python's ec module) to generate a curve key pair, sign a message, and verify the signature.

Python Example

from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.asymmetric import ec

# Generate key pair
private_key = ec.generate_private_key(ec.SECP256R1())
public_key = private_key.public_key()

# Sign message
message = b"Hello, ECC!"
signature = private_key.sign(message, ec.ECDSA(hashes.SHA256()))

# Verify signature
try:
    public_key.verify(signature, message, ec.ECDSA(hashes.SHA256()))
    print("Signature verified!")
except:
    print("Signature verification failed!")
